# SYSTEM BEAUTY: Архітектура та Внутрішня Логіка Проекту

Ця документація створена для майбутнього використання ШІ або розробниками, щоб миттєво зрозуміти структуру, стек та ключові рішення проекту, і вносити зміни без порушення продуктивності та наявної логіки.

---

## 1. Загальний огляд
- **Назва:** SYSTEM BEAUTY / BLESS & YEEZY Event Landing Page
- **Тип додатку:** Single Page Application (SPA), Landing 
- **Головна мета:** Презентація преміальної бізнес-освіти/іВенту для б'юті-індустрії.
- **Дизайн Код (Aesthetic):** Різкий, висококонтрастний "Instagram event" стиль. Глибокий "абсолютний" чорний фон (`#000000`), яскраві неоново-жовті акценти (`#FACC15`), суцільні жовті бейджі з чорним текстом, жирна та щільна типографіка (Unbounded).

## 2. Технологічний стек
- **Ядро:** React 18, Vite
- **Стилізація:** Tailwind CSS (включає гнучку систему кастомних кольорів у `tailwind.config.js`).
- **Скролінг:** Lenis (`lenis/react`) — алгоритм плавного "шовкового" скролу сторінки (Smooth Scroll).
- **Анімації:** GSAP (GreenSock) + ScrollTrigger — наймогутніша бібліотека для скрол-анімацій на ринку.
- **Роутинг:** React Router DOM (Наразі має '/' для головної та '*' для сторінки 404).
- **Іконки:** Lucide React (`lucide-react`).
- **Деплой:** Vercel (використовує файл `vercel.json` для правильної конфігурації SPA routing, щоб працювали прямі посилання на неіснуючі шляхи типу 404).

## 3. Ключова Архітектура (Компоненти)
Вся лендінгова логіка зібрана в `App.jsx`, який є композицією менших функціональних блоків:

### 3.1. Візуальні "Цукерки" (Micro-interactions)
*   **`Preloader`:** Приховує сайт (встановлює `opacity: 0`), поки не завантажаться шрифти (`document.fonts.ready`). Дає перше враження "дорогого продукту" з анімацією логотипу пульсуючим неоном.
*   **`CustomCursor`:** Кастомний жовтий флюїдний курсор.
    *   *Архітектурне рішення:* Використовує маніпуляції напряму через `useRef` та `requestAnimationFrame` замість `useState`, щоб обійти React-цикл рендерингу та видавати кристально плавні 100+ FPS без затримок.
    *   *Безпека:* Автоматично відключається на Touch-девайсах (`window.matchMedia('(pointer: fine)')`), щоб не ламати iOS/Android взаємодію.
*   **`ScrollProgress`:** Смужка нагорі сайту. 
    *   *Архітектурне рішення:* Взаємодіє з даними від Lenis та використовує `transform: scaleX()` замість зміни `width`, спираючись на GPU-прискорення.
*   **`TiltCard`:** Логіка "голографічних/3D" карток, які повторюють рухи мишки на десктопі.

### 3.2. Навігація
*   **`Navigation`:** Sticky Header (Липке меню), що стає видимим тільки після прокрутки стартового екрану повз "hero" секцію. 
    *   Використовує `IntersectionObserver` для відслідковування того, на якій саме секції знаходиться користувач, та підсвічує відповідну кнопку зверху і на мобільних прогрес-барах.

### 3.3. Ключові Секції (`App.jsx`)
*   `BackgroundEffects` — Анімовані жовто-сині плями на задньому фоні (CSS Blob).
*   `Hero` — Відкриваючий екран з жирною типографікою та бейджами.
*   `Reality` — Проблематика ринку.
*   `MasterTrap` — Масштабований круговий фон через `GSAP.to` (круговий градієнт), який змінює масштаб `scale: 1.5` при прокрутці.
*   `TheProblem` — Акцентна секція.
*   **`HorizontalCamps` (Секція Архітектури):** Найбільш складна частина.
    *   *Логіка Desktop:* Використовує GSAP `СcrollTrigger: ping: true`, щоб зупинити вертикальний скрол і рухати картки горизонтально, розраховуючи `scrollWidth`.
    *   *Логіка Mobile:* GSAP адаптовано через `gsap.matchMedia()`. На мобільних (до 768px ширина) відключається "pin" скрол. Натомість використовується нативна технологія `overflow-x-auto snap-x`, і вмикається підказка "ГОРТАЙ ВБІК" (Swipe Indicator).
*   `Results` — Фінальні бонуси програма.
*   `Footer` / Контакти.

## 4. Сторінка `NotFound` (404)
Модульна, багата інтерактивна сторінка `NotFound.jsx`. Має величезний анімований текст 404 з градієнтом, який слідкує за мишкою (режим відслідковування курсору), фонові ефекти Blob, та кнопку повернення "До системного бізнесу". 
Логіка входу анімується плавно за рахунок мікро-таймера (`setTimeout(..., 10)`) у `useEffect`, усуваючи повідомлення ESLint про синхронний каскадний рендер.

## 5. Правила майбутньої розробки (Правила для ШІ):
1.  **Продуктивність понад усе:** Всі швидкозмінні анімації на скрол та рух мишки (CustomCursor, ScrollProgress, Hover Tilt) МАЮТЬ використовувати `transform` або `opacity`. Заборонено робити анімацію через `width/height` чи записувати координати прямо у `useState` (швидко викликає "lag" рендеру Reactу).
2.  **Мобільна адаптація:** Щоразу, як додається GSAP "Pinning", перевіряти, як це впливає на мобільні. Надавати перевагу `snap-x` CSS та нативному скролу на мобільних.
3.  **Витримати стиль:** Жодних блідих чи пастельних кольорів у дизайні (окрім сірого тексту `textMuted`). Акценти повинні бути чисто чорні або чисто жовті (`#FACC15`) з високою помітністю "в лоб" для подібності афішам BLESS & YEEZY.
4.  **GSAP Resize Issues:** Всі `ScrollTrigger` кінцеві точки (end/start) які базуються на розмірі елементів (наприклад ширина горизонтального скролу), повинні обчислюватися ДИНАМІЧНО як функція `end: () => ...` і супроводжуватись `invalidateOnRefresh: true`, щоб уникнути багів при зміні розміру вікна браузера чи повороті планшету.
